### 2026-03-04 21:42

**OpenClaw Agent 创建流程**

1. **创建 workspace 目录**
   ```bash
   mkdir -p ~/.openclaw/workspace-<agent-id>/memory
   ```

2. **创建 workspace 文件**
   ```
   workspace-<agent-id>/
   ├── SOUL.md          # 身份定义
   ├── AGENTS.md        # 工作空间说明
   ├── USER.md          # 用户信息
   ├── MEMORY.md        # 长期记忆
   └── memory/          # 每日日志
       └── YYYY-MM-DD.md
   ```

3. **创建 agent 配置目录**
   ```bash
   mkdir -p ~/.openclaw/agents/<agent-id>/agent
   mkdir -p ~/.openclaw/agents/<agent-id>/sessions
   ```

4. **复制 auth-profiles.json 和 models.json**
   ```bash
   cp ~/.openclaw/agents/main/agent/auth-profiles.json ~/.openclaw/agents/<agent-id>/agent/
   cp ~/.openclaw/agents/main/agent/models.json ~/.openclaw/agents/<agent-id>/agent/
   ```

5. **更新 openclaw.json**
   ```json
   {
     "id": "<agent-id>",
     "name": "<agent-name>",
     "workspace": "/root/.openclaw/workspace-<agent-id>",
     "agentDir": "/root/.openclaw/agents/<agent-id>",
     "sandbox": {"mode": "off"},
     "tools": {"allow": ["*"]}
   }
   ```

6. **重启 Gateway**
   ```bash
   openclaw gateway restart
   ```

**目录结构总结：**
```
~/.openclaw/
├── workspace-<agent-id>/    # agent 的 workspace（SOUL.md 等）
└── agents/
    └── <agent-id>/
        ├── agent/           # auth-profiles.json, models.json
        └── sessions/        # 会话数据
```

**Solution Agent 工作流程定义**

5阶段多模型协作流程：

1. **Phase 1: 多模型并行方案生成**
   - Claude Opus 4.6, Claude Sonnet 4.6, GLM-5, GPT 5.x
   - 各自独立输出技术方案

2. **Phase 2: 自我 Review（2轮）**
   - 各模型对自己的方案 Review → 完善
   - 至少重复 2 次

3. **Phase 3: Claw 合并 Review**
   - Claw 收集各模型方案
   - 对比分析、合并产出 Review 版

4. **Phase 4: 交叉 Review（3轮）**
   - 合并版交给各模型 Review
   - 至少重复 3 次

5. **Phase 5: 最终输出**
   - Claw 最终整合
   - 输出到 SOLUTION.md
