# .claude 目录协作规范

本目录遵循“高质量定义 + 自动调用”原则：

1. `agents/`：定义角色能力边界（YAML 头 + 详细说明）
2. `skills/`：定义可复用任务能力（输入、输出、约束）
3. `hooks/`：定义执行阶段自动检查（如文档沉淀、测试门禁）
4. `commands/`：定义可复用命令入口

## 执行原则
- Claude Code 根据目标自动识别并调用上述能力，不要求流程层硬编码调用顺序。
- AIFD 只负责给出阶段目标、输入/输出、质量门禁和审批节点。
- 每次阶段结束后，必须完成 docs 沉淀检查与 agent-memory 回写。

## 定义质量要求
- 前半段使用 YAML frontmatter（name/description/version）
- 后半段包含：职责、输入输出、执行清单、交付标准、常见风险
- 动态内容使用“业务领域要求（项目初始化注入）”整块注入
